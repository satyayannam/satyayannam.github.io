Biometric Identity Verification using Palm-Line Topology

This project is a research-driven initiative to create a palm-line based biometric recognition system using computer vision, feature engineering, and privacy-first security protocols. Unlike facial or fingerprint biometrics, palm-line patterns are less intrusive, harder to replicate, and ideal for sensitive domains like healthcare and defense.

🔍 The Problem We're Solving

Facial recognition systems have become widespread — but they come with high risks: spoofing attacks, surveillance misuse, and identity theft. Our approach replaces facial data with palm biometrics, which are contactless, private, and cryptographically securable through keypoint hashing.

🧠 Technical Stack

  • Python + OpenCV: Image capture, contour filtering, palm line segmentation
  • Keypoint Detection: ORB (Oriented FAST and Rotated BRIEF) descriptors to extract unique line junctions
  • Contour Mapping: Convex hulls, ridge line curvature, and geometric normalization
  • Feature Matching: BruteForce + Hamming Distance (ORB) and FLANN (optional)
  • Anti-Spoofing: Liveness check based on hand movement + depth map estimation (Planned)
  • Biometric Hashing: SHA256+ feature vectors to create anonymized identity signatures
  • Frontend: HTML/CSS with WebRTC camera support (in-progress)

📐 Process Pipeline

  1. 1️⃣ Capture palm image using webcam
  2. 2️⃣ Convert to grayscale + apply adaptive thresholding
  3. 3️⃣ Detect contours and extract regions of interest (ROI)
  4. 4️⃣ Use ORB to find keypoints and descriptors on palm lines
  5. 5️⃣ Match descriptors with known hashed signatures for verification
  6. 6️⃣ Display identity or deny access based on similarity threshold
